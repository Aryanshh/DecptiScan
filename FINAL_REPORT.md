## 1. Executive Summary
DeceptiScan is a state-of-the-art fraud detection ecosystem designed to identify deceptive job postings and offer letters. By combining rule-based heuristics, machine learning (BERT), and human-level AI interpretation (Mistral), the system provides a comprehensive "Confidence Index" for every scanned document.

## 2. Technical Architecture

### 🛡️ Frontend (User Intelligence Interface)
- **Design System**: Advanced Glassmorphism with real-time backdrop saturation and blur.
- **Motion Engine**: GSAP-powered staggered entrance and elastic result reveals.
- **Input Methods**: Binary file upload (PDF/Word) and raw text sanitization.

### ⚙️ Backend (The Bridge)
- **Environment**: Express 5.x with a custom regex-literal router for 100% path compatibility.
- **N8n Integration**: Secure binary forwarding using `form-data` and `multer` memory buffers.
- **Mock Model**: Built-in fallback prediction engine (Port 3001) for zero-downtime operation.

### 🧠 Intelligence Layer (N8n + ML)
- **Heuristic Engine**: 50+ keyword banks targeting financial red flags, urgent phrasing, and fake authority triggers.
- **BERT ML Service**: Standalone Python FastAPI service using `distilbert-base-uncased` for statistical fraud probability.
- **AI Agent (Mistral)**: Contextual analysis to detect poor grammar, unverified domains, and unrealistic bait offers.

## 3. Security Analysis Model

The system uses a **Triple-Verdict Alignment** strategy:
1.  **Layer 1 (Statistical)**: ML Model detects patterns of fraud unseen by simple keywords.
2.  **Layer 2 (Linguistic)**: Heuristics flag specific high-risk terms (e.g., "Training Fee", "Security Deposit").
3.  **Layer 3 (Contextual)**: Mistral AI interprets the intent and professionalism of the document.

*A "Fake" verdict is triggered only when the Weighted Hybrid Score exceeds 75% across these layers.*
